Design-decision summary

  1. Hero with typography hierarchy, not cards. Small uppercase label → big name → status row. Matches Linear/Stripe/Notion. Key actions always visible, right-aligned.
  2. "At a glance" 4-up stat row. What a farmer needs within 2 seconds of opening the page: days, incubator temp, humidity, room temp. The ambient one is amber with "Ambient" badge — instantly readable as "different kind of info."
  3. Progress timeline visualised. Current day marker, lockdown, hatch day all on the same track. Replaces the bare progress bar.
  4. Section anchors. Major modules get 4px gold left border + small uppercase "Section 01" label + large heading. Impossible to miss where a section starts. Ambient sections get the same treatment with an amber accent.
  5. Generous whitespace. mt-16 between major sections (was mt-10). p-6 to p-8 on cards (was p-5).
  6. Tabs for the detail pile. Daily log, Milestones, Timing, Egg weights, Raw readings — all behind tabs. Default tab is the one farmers check daily. Keeps the page short; power users drill in.
  7. Amber discipline for all ambient UI. Same amber tokens everywhere (dashboard card, hero card, sensors subsection, daily log columns, banner). Once a user learns "amber = room" they never re-learn.
  8. De-emphasised metadata lives below. Hatch details (breed, source, incubator model) go in a lighter section at the bottom — no section anchor bar — because you read it once at hatch start and rarely after.
  9. Status badge in traffic-light convention. Green "Active" with glow dot. Green/amber/red already used for sensor health. Consistent.
  10. Subtle card hover. Border brightens + 1px lift + soft shadow. Tells users cards are clickable without being loud.
